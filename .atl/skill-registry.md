# Gentleman-Skills Registry

This document registers the locally loaded skills for the DVGE project. These skills are globally installed in `~/.gemini/skills/` and are available to enforce the Code Style, Architecture, and specialized niches per project requirements.

## 🎨 UI/UX & Frontend
- `3d-web-experience`: Specialized in rich 3D interactions.
- `canvas-design`: Canvas and visual assets generation.
- `scroll-experience`: Scroll interactions and animations.
- `ui-ux-designer`: Standard UI/UX logic.
- `ui-ux-pro-max`: Premium UI/UX logic and layouts.

## 📝 Documentation & Technical Writing
- `api-documentation-generator`: REST/GraphQL API docs.
- `code-documentation`: Inline code comments and function docs.
- `documentation-engineer`: Architecture and system-level docs.
- `documentation-expert`: Standard process documentation.
- `readme-generator`: Markdown generation for root README.
- `technical-writer`: End-user guides and SDK documentation.

## 📊 Analytics, Reports & SEO
- `report-generator`: General Arsenal/Skill reporting.
- `report-writing`: Formal reporting structures.
- `seo-analyzer`: SEO auditing.
- `seo-optimizer`: On-page SEO strategies.
- `screenshot-business-analyzer`: Analyzes screenshots for business context.
- `screenshot-interaction-analyzer`: UX interaction states from images.
- `screenshot-reviewer`: QA reviews of visual assets.
- `screenshot-synthesizer`: Combines/extracts UI structures from screenshots.
- `screenshot-ui-analyzer`: Detailed component analysis.

## 🛠️ Development, QA & Other Nicknames
- `changelog-generator`: Generates semantic release notes.
- `diagram-architect`: Mermaid and system diagrams.
- `legal-advisor`: Open-source licensing and legal disclaimers.
- `peer-review-assistant`: PR and code review standards.
- `coverage-analysis`: Unit testing and coverage reports.
- `copywriting`: Marketing text and branding.
- `affiliate-blog-builder`: Specialized blog generator.
- `xlsx`: Excel and spreadsheet data manipulation.
- `scripts`: Utilities and build tasks.

---
> **Note to AI**: Always refer to the corresponding global skill in `~/.gemini/skills/<skill-name>/SKILL.md` before starting a task in these niches.
